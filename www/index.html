AIC

A local Context Bridge for Large Language Model prompting. AIC watches your files, compiles context, and prepares prompts automatically.

Overview

AIC is a background tool designed to streamline the workflow of copying code and context into AI chat interfaces. Instead of manually copying files, running shell commands, and pasting them into a browser, AIC automates the process.

It watches a specific file (.aic/prompt.md). When you save that file, AIC:

  1. Executes defined shell commands (optional).
  2. Reads your current system clipboard.
  3. Collects global project rules.
  4. Combines everything into a structured prompt.
  5. Writes the final result back to your clipboard, ready to paste.

Project Structure

When initialized, AIC creates a dedicated directory in your project root:

./.aic ├── prompt.md <-- You write here. The "Watcher" monitors this. ├── rules/ <-- Text files placed here are auto-appended to every prompt. └── prompts/ <-- History. Past prompts are stashed here automatically.

Usage

To use AIC, simply edit the .aic/prompt.md file. The file uses a specific format separated by ---.

The Prompt Format

---
# Section 1: Pre-processing Commands (Shell)
# Commands here run BEFORE the prompt is built.
ls -la ./pkg
tree ./src
---
# Section 2: The Prompt Body
# This is what you actually want to ask the AI.

How it processes

When you hit Save on prompt.md:

  1. Clipboard Capture: AIC grabs whatever is currently in your clipboard (useful if you just copied a stack trace or a specific function).
  2. Rule Injection: It reads all files in .aic/rules/.
  3. Output: The final text placed in your clipboard looks like this:
[Content originally in your clipboard]

=== RULES ===
FILE: .aic/rules/coding-standards.txt
[Content of rule file]

=== PROMPT ===

Key Features

Auto-Stashing

You never lose a prompt. Every time a prompt is processed, the raw content is saved to .aic/prompts/YYYYMMDD_HHMMSS.md.

History Pruning

AIC maintains a clean history, automatically keeping only the last 100 prompts to save disk space.

GitIgnore Aware

When collecting files or traversing directories, AIC respects your project's .gitignore file to avoid leaking sensitive or build artifacts.

Debounced Watcher

The file watcher includes intelligent debouncing to prevent double-triggers or processing partial writes.
